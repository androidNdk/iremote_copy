About

Remote PC Suite is a host application that allows you to connect your devices, running iRemote Suite software, and manipulate your desktop or notebook.

iRemote Suite 1.4 features:

  • Desktop screen sharing with zooming support. This allows you to observe either complete screen or a part of it, as close as it is necessary;
  • Mouse control (mouse movement, mouse clicks, tap-to-click, scrolling);
  • Keyboard control;
  • Master volume control;
  • Fully-functional shell access. You are able to control you PC/Mac with the help of shell commands;
  • Authorization feature. This feature allows you to remember your device once, so you do not have to approve each connection on the desktop.
  • iOS and Android devices are supported.

Supported desktop platforms:

  • Mac OS X 10.5 or above (Intel only);
  • Windows XP / Windows 2003 Server / Windows Vista / Windows 7.

Definitions

Remote PC Suite - is desktop software. You are always able to download latest version of the tool at http://www.scientific-soft.com/mobile/apps/iremote/.

iRemote Suite - is mobile application for iPhone. You have to proceed to Updates section of App Store application on your iPhone to check whether there are any updates of iRemote Suite. Note, that iRemote Suite must be installed on your iPhone.

1. Managing Connections

The most wide spread way to control you desktop is being inside the same local network (to be connected to the same WiFi router). When both devices (desktop and mobile phone) are connected to the netwotk, you can rpoceed to launching Remote PC Suite/iRemote Suite.

You will see the following confirmation dialog when anyone attempts to connect to your computer.

The following options are available at the moment:

  • Allow connection once.
  • Deny connection.
  • Allow connection and remember device.

If you want to remember this device for future use, select 'Allow & Remember' option. You will be immediately prompted for the password.

Note: You are only allowed to enter password using primary keyboard on the device.

Then you will be prompted to enter this password on the device.


Authenticating the device.

Here you need to enter the same password that you've just used on the desktop.

After this procedure you device becomes authorized and you will not have to allow connection explicitly on the desktop each time.

Note: RemotePC allows the only device running iRemote Suite to be connected at the time.

You may disconnect device clicking “Disconnect” button on main application window.


Part of Remote PC main window with “Disconnect” button highlighted.

2. Managing Users

You may access users manager by clicking corresponding button on the main application window.


Part of Remote PC main window with “Users Manager” button highlighted.

Here you are able to perform the following operations

  • Edit device.
  • Remove device.

Editing allows you to set new password and change IP policy. Each account is being created with strict one by default. This means that a device is associated with certain IP address. Here you can chenge this ip and or disable this option at all.

Note: It is highly recommended to keep this option enabled due to security considerations.

3. Configuring the Application

Spreading screenshots

This group of settings allows you to enable or disable sharing screenshots with connected devices. You are also able to specify frame rate. This value may be float. For example, it allows you to grab and spread one screenshot per 4 seconds, by specifying value of 0.25.

Port value

Sometimes you have to change port number when another software uses 8444 value, which is default for Remote PC.

Note: When you apply port changes, application restarts service, and all currently connected clients will be disconnected.

Possible Errors

  • Server has failed to start. Change port value in application settings.

    This error occurs, if you specify port value, which is already busy. In this case, you should specify another port value. If you are sure, that specified value is correct, try to apply it ("Settings"->"Apply"), until this error appears.

    Note: When you see this message, server is not running. There are two ways to start it. The first on is to quit application and launch it again. The second one is to apply settings ("Settings"->"Apply").

4. Restrictions and Limitations

  • Application does not simulate DirectInput events.
  • You may connect the only device, running iRemote Suite, to your computer at the time.
  • Application settings are stored in Config.xml file. You should not modify this file directly. Use application UI for such purposes.
